【主题】Nostr 爆火后，去中心化社交才刚刚开始

【核心内容简介】

Nostr 是種不基於區塊鏈的去中心化社交協議，2.1 因 Jack Dorsey 宣佈基於 Nostr 開發的 Damus 受大量關注

Nostr 特色是透過公私鑰進行認證、創建賬戶，理論上能支持所有區塊鏈。若說 Coinbase 高管 Dan 創建的 Farcaster 是創新的以太坊，Nostr 則是遵循簡單、充滿密碼龐克精神的比特幣

去中心化社交立意良善，而協議缺乏激勵機制，不過 Nostr 開發這回復：中繼端運營者不傻，不假設無償服務，他們是有自己目的

中繼端不保證數據存儲、且用戶更換客戶端數據也會消失。未來或針對存儲進行激勵

【个人思考】

去中心化社交是我的資訊盲區，也是我刻意想迴避的領域。

一直以來有意識的避免使用過多社群軟體，雜亂、但又會讓你成癮的演算法推薦，蠶食著大量注意力，是我很困擾的罪惡感來源之一

大多時候，我並不在意個人資料的導出、遷移，但當每當在社群上被洗版特定議題、惱人的投資廣告、詭異的商品推薦，都會令我反感，然因為是主要聯絡方式，又積累大量屬於自己創建的資訊，實在脫離不了

又是在想創建新賬號時，又開始在意歷史貼文、社交資訊及有想要保留的部分

才意識到，資料所有權在自己手上，能任意遷移至其他平台，是件多麼可貴的事情；儘管當前幾乎沒有人在意

開腦洞：數據的導出、遷移，本質上是一種資料結構的轉換，現在網路上各種免費轉檔網站，靠的是廣告費；將來或有一種應用、協議，是針對不同社群平台開發，能幫你導出歷史資料，轉至適合新協議的格式

【输出人】wenchuan
【主题】Fuel：模块化执行层和 L2 有何不同？
【核心内容简介】Fuel，和 Celestia 有同一个联合创始人，与 Celestia 形成了互补关系，其定位是模块化执行层。Fuel 试图通过模块化的执行层来优化基础层，并且改进 L2 的执行状态，真正实现扩容。
Rollup 因对 DA 没有改进，导致 gas 费还是会受主网拥堵的影响，此外，还需要支付存储费，但是目前 Rollup 对状态优化的关注并不多。作为 celestia 的互补，Fuel 认为状态和执行会成为新的瓶颈，故而提出模块化执行层。
Fuel，通过轻客户端和欺诈证明解决可扩展性的问题；通过新的虚拟机 FuelVM 改进了 EVM 低效、浪费资源的执行环境。
Fuel，可以作为 L1、 Rollup 或 L2、状态通道以及侧链。
技术上，Fuel 可执行并行交易、开发了新的编程语言 Sway、并使用新的虚拟机 FuelVM。
【个人思考】模块化的概念被大家熟知以后，出了不少项目，Cevmos 针对结算，Fuel 针对执行层，针对主权 rollup 的有 dYmension 、Eclipse。Fuel 的定位是先行一步，假设了带宽会很便宜，计算的吞吐量更为关键，具备前瞻性；而 FuelVM 本身就具备防欺诈可以更好地利用硬件提高交易执行的吞吐量，也是 Fuel 定位的实践。其次，作为专注执行层的模块，让人不由得想到去年的明星项目，Altlayer，一个定位特定应用的执行层。一个自下而上，一个自上而下，产品逻辑迥然不同，值得关注二者后续的发展路径。

【个人思考】
NFT 市场的洗售行为除了为撸平台空投代币的激励，更多是为了拉高价格或者拉高需求市场以骗取买盘。本文最值得推荐的部分是第四节，通过三层过滤器，从数据上解决如何发现和过滤洗售交易数据。这样的思维方式非常值得学习，从数据的角度分析市场，去伪存真。

【输出人】@猫老大
【主题】为什么 SEC 对加密质押商痛下杀手？
【核心内容简介】
2 月 10 日，Kraken 与美国证券交易委员会（SEC）达成和解，支付 3000 万美元罚款并将终止面向美国用户的质押服务。为什么 SEC 要重手整治 Karken 的质押服务？Karken 又有哪些把柄被 SEC 拿住，不得不支付巨额罚款和终止服务？本文带我们一起进行原因的解读。
SEC 投诉的关键论点是，与客户自己可以实现的收益相比，质押服务和质押资产池提供了优势和更高的回报，这在本质上将通过 Kraken 进行质押的用户，与自己独立质押的用户区别开来。
Kraken 质押服务参与者的收益最终由 Kraken 决定——而且这些决定是不透明的且由 Kraken 按照自己的时间表进行支付。同时，也并不是所有存放在服务中的代币都被质押了，即 Kraken 本身可以挪用这些资金。
这两个问题可能导致了 SEC 对 Kraken 的最终裁决。
最后，译者观点，SEC 对 Kraken 的重磅出击，不能等同于对整个加密货币的攻击，因为链上的直接质押目前没有受到影响。
【个人思考】
最近几天 SEC 对 Kraken 的重磅出击以及 SEC 主席不停的放话，导致之前势头不错的市场应声下跌。当然下跌的原因可能有很多，但政府对加密监管的出击也占据很大一部分因素。
不过在这里，我们不讨论监管方面。我想说，还记得那句话吧，“如果你不知道收益怎么来的，你就是那个收益”，昨天我看到 coindesk 一篇文章，指出 Kraken 是第四大加密货币交易所，它的质押服务收益每年高达 24%！！！这个太高了，尤其在目前的熊市阶段，并且这种收益一点都不透明。
在阅读本文之后，我更是觉得，刚经历过 FTX 的糟蹋，目前 SEC 对 Kraken 处理一下也不算坏事。当然，加密行业目前正处于美国加密监管的枪口之下，市场有风险，还需谨慎。

【核心内容简介】
本文从自上而下的逻辑拆解，分析行业内的增长究竟在哪里，以寻找确定性的机会，最底层是公链效率 → 开发者数量 → 应用数量 → 用户数量 → 市场规模。
市场规模：依赖于币价与用户规模。
用户规模：Gas 消耗（有真实需求在被满足）、活跃钱包数量（有人在用）
合约规模：TOP10 创建合约的地址创建了 49% 的智能合约，新创建的合约中拥有稳定用户群的数量非常少。
开发者规模：稳步增长，但 2022 下滑。2022 年中开发者贡献数量打破了过去只有年底才会下降的预期，有了较为持续的下滑，同时只有 30% 的核心开发者在第二年会继续贡献代码。
公链吸引的开发者：以太坊还是毫无疑问拥有最多核心开发者的公链，波卡次之，Cosmos 在 2022 年超过了比特币成为了核心开发者数量第二大增速最快的链，Solana 画出一条过山车式的增长曲线，Polygon 的增速在 2022 年还有接近 100% 的增长。
【个人思考】
可以重点关注 Buidler 数量指标，本文定义“每月贡献 50 次 commit 并且坚持了超过 50% 时间的人”为真信仰的开发者，这一数据正在稳步增长，但是 2022 年增速降为 0，需要注意这一市场情绪是否将长期存在或者恶化。作者归纳的两个确定性赛道与常识基本符合，一是开发者服务如 Wagmi、Alchemy，二是优化交易行为如 GMX、Blur。但需要注意的是，投资视角与创业视角有很大不同，确定性赛道意味着内卷，如何笃定正确但不确定的增长方向，才是下个周期的财富密码。

【核心内容简介】
本文针对集成 ChatGPT 的 Bing 搜索给出比较全面的评测。
集成了 ChatGPT（ 据传是定制改进过的版本 ）的 Bing，名字叫 “New Bing”，与老 Bing 相比，会在搜索框下面提醒你试一试问搜索引擎一个具体问题。当随便搜索一个问题后，拿鼠标滚轮往上一滑动，进入一个与搜索引擎对话的界面。
与 ChatGPT 单纯的给答案不同，New Bing 还会给你列出它所生成答案的所有信息源。精准命中你的问题，并且有理有据地给出信息源。对搜索引擎来讲，这是一种革命性的体验提升。除了效率的提升，相较于 ChatGPT 免费版陈旧的数据库，New Bing 是具有时效性的，它可以给到你新的信息。
经过一系列对话，可以判断：现在的 New Bing，不仅是具有时效性的，它还会结合各类因素，做一定程度的类似于 “ 思考 ” 的动作。New Bing 在实用性方面，要远远高于前些天大家试用的 ChatGPT 。 New Bing 能回答的问题的范围限度也比试用版的 ChatGPT 要高很多。同时，与 ChatGPT 的嘴硬相比，New Bing 会修正自己的错误，而且是有理有据的修正。
【个人思考】
集成 ChatGPT 的 Bing 搜索，大大的提升了搜索体验，用户不再是自己从网页中自行翻找有效答案，New Bing 会列出它所生成的答案及其所有信息源。不过准确性还有待提高。
本文评测中的各种精彩互动，不得不感叹 AIGC 的强大。AI 神经网络的算法逻辑与传统的搜索爬虫抓取的逻辑完全不同，呈现方式也不同，更具有连贯性和符合人脑思维逻辑，这种方式很可能会打破搜索引擎的信息垄断，同时改变目前搜索广告获利的方式。还有，随着准确性的提高，New Bing 的这种搜索体验可以提供靠谱的信息筛选，从而影响以往攻略类、种草类的产品。
另外，这一次较量，Google 有点落后，而且在中文搜索方面 Bing 这几年有大大的提升也一直没有放弃过中国市场。感觉一系列思考之后，新一轮的产品又随着新技术的应用要开始洗牌了。

【主题】数据可用性层（DA）如何塑造区块链的未来
【核心内容简介】
1. 什么是数据可用性：数据可用性是指节点对区块链网络中交易数据进行查看和下载的需求。数据可用性最初并不是一个问题，只是在扩容跨越了一定的门槛之后，数据可用性才成为一个不可避免的话题。数据的可用性与 Rollup 的安全性密不可分。
2. 数据可用性的发展和实施路径：逐步分离数据可用性、实现独立的数据可用性层、以太坊如何验证数据可用性、数据可用性层的经济模型和市场潜力
3. 区块链的数据可用性和模块化：DA 与主网分离的历程，就是区块链的模块化，这是一个更长期的趋势。执行层、DA 层分别模块化后，区块链主网唯一需要承担的就是共识。。至此，一个 "共识-执行-数据可用性 "的结构就形成了，这个结构是相互独立的。
【个人思考】完整地介绍了 DA，从逻辑上一路捋下来，清晰、明了。对于独立的 DA，还有与之相竞争的两个方案，一个是 Dansharding，这个目前可以关注以太坊自身以及 Near；一个是 Rollup 自身随数据结构和费用提出的 EIP-4844，以及 EIP4488。可以看到在终局还未明确的状况下，技术的发展及其竞争态势，DA 只是一个小的方面，放大来看，其实整个区块链都处于这个阶段。DA 的这个过程，让我更加体会到：各种层出不穷的技术，都是针对主要技术的关键发展方向开展各类打补丁的行为。终局还没确定，持续关注各类”补丁“技术的发展以及观察是否有从天而降的黑马项目。

【主题】详解 MPC 和智能合约钱包的优缺点与面临的挑战
【核心内容简介】
自我托管一直被誉为管理加密资产的最佳实践。相比传统（HD）钱包，有两类替代密钥管理解决方案取得了重大进展：智能合约钱包 （ 包括多重签名钱包 ） 和多方计算 （MPC） 协议。
1. MPC 钱包
多方计算 （MPC） 使一组互不信任的各方能够根据他们的输入共同计算一个函数，同时保持这些输入的私密性。
优势：无单点故障、可调整的签名方案、更低的交易和回收成本、区块链不可知论者
缺点：链下问责制、与许多用户采用的大多数传统钱包不兼容、大多是孤立的定制产品
值得注意的技术：可编程密钥对
2. 智能合约钱包
智能合约钱包是一种行为类似于钱包的智能合约，即一个允许用户管理资金、进行 web3 登录和与 dApp 交互的界面。多重签名钱包是智能合约钱包。
优势：无单点故障、可编程访问控制、可扩展、可编程恢复、链上问责制、支持迁移到其他签名方案、开源
缺点：更高的费用、没有得到普遍支持、恢复成本更高、与不可升级的合约不兼容。
值得注意的技术：账户抽象
3. 钱包开发生态系统面临的挑战
技术漏洞、社交攻击层面、安全和迁移成本、操作安全
结论：尽管 MPC 和智能钱包有着共同的「这个 vs 那个」框架，但从长远来看它们并不是竞争关系，而是互补关系。
【个人思考】作为钱包这种基础生态，本质上安全性和强认证特性是最重要的，但是 metamask 这类 HD 钱包能够成为目前的龙头，很大程度上也就是其易用性，目前开始讨论 MPC、智能合约钱包等，也就说明该赛道的竞争进入了另一个阶段。MPC 与智能合约钱包的区别，也就是一直争论的多签 vs 多方计算。除了文内提到的 MPC 的缺点，MPC 还缺少同行评审，而且缺少 HSM 支持；而多签针对不同的 L1 需要实现不同的智能合约，复杂程度较高。二者的优劣明显，基于此个人很认可文章提到的二者会形成互补关系，甚至可能是该赛道的终局。

对于 NFT-FI 三难困境的破除、新的流动性解决方案和金融化探索的需求也从未减弱，NFT 一站式平台 Furion 结合了镜像通证、指数通证、AMM、点对池借贷等功能给出了自己的答案。

镜像通证：NFT→F-X Token & 指数通证：FFT

（1）单独的 NFT 标的流动性池

Furion 为每个 NFT 标的提供了单独的流动性池，用户可以在相对应的流动性池子中存储或锁定他们的 NFT 并铸造得到 F-X Token 以此作为该 NFT 在 ERC-20 协议中的镜像通证，存储的 NFT 可以用 F-X Token 销毁赎回，锁定的 NFT 在锁定期内可以赎回，锁定期外将会释放到公共的存储池并收取惩罚。

（2）聚合类似但不同的 NFT 标的流动性池

聚合池可以把多个 NFT 标的的镜像通证作为一个池子，以聚合不同的但类似的 NFT 标的的流动性（项目方是共同一家的或者是某个项目的多个仿盘）。比如以 Yuga Labs 旗下的 NFT 标的聚合为池子，可能包括 F-BAYC、F-MAYC、F-BAKC 等。用户把他们的 F-X Token 存入到聚合池并铸造得到 FFT。FFT 是有一揽子 NFT 支持，并代表着所组合 NFT 标的的指数。通过销毁 FFT，用户将能重新取回其组合的 F-X Token。

针对 F-X Token、FFT、FUR 作 AMM 交换

当把 NFT 变为 ERC-20 镜像通证后，与一贯的 AMM 交易网格类似，Furion Swap 也以每两个 Token 组合而成一个交易对，然后遵循 x*y=k 的公式有一般的交易者和 LP 提供者。

点对池（Peer-to-Pool）借贷

用户可以通过借出代表不同 NFT 或一揽子的 F-X Token、FFT 或者其他 ERC-20 通证来赚取利息收入，或者将其作为抵押品后进行借款。

Furion 预言机规则

Furion 有一套自己的预言机规则，通过收集所有 NFT 市场的历史交易来构建 NFT 的价格曲线，简单而言，即是在限定时间内发生的 N 笔交易历史中，有大部分的交易价格落在某个价格区间内，那么便取该价格区间的下限作为 F-X Token 的参考价格。

GNS 的机制：
简单来说其为一个去中心化永续合约平台：
1、预言机定价，LP 和 Trader 对赌
2、LP 为纯稳定币，支持 外汇/ 股票/ 加密货币 交易
3、双向资金费率，跟 CEX Perp 一样一方向另一方支付费用
GNS 在交易侧和 LP 侧各设有三重机制控制风险，其核心为：
1、资产现货流动性决定场内交易滑点，防止价格操纵
2、资产价格波动性和多空比决定持续持仓成本，应对单边行情
3、净值模式加上流动性调节以及现金流循环，构建稳健的 LP
GNS 使用了 Spread， Rollover Fee， Funding Fee 三重机制来进行交易侧的风险控制。
1、Spread: 额外的开仓费用，开仓头寸越大，资产流动性越差，费用就越高。用于预防价格攻击，便于上架小币种。
2、Rollover Fee: 现货波动率来定价，用于控制交易者的杠杆和风险
3、Funding Fee: 多空头寸之差和现货波动率来定价，多/空>1 时多头支付空头，反之亦然，用于平衡多空比，避免单边敞口过大。
竞争优势：
核心优势就是在通过其复杂的风控机制，提供了一个体验合格的外汇/股票衍生品交易场所，在这些资产上交易体验独占鳌头，让其产品能够立的住。另一方面双向资金费率等又让其和 GMX 实现了差异化竞争，在加密货币方面也成功获取了部分客户。做到这些离不开 GNS 团队的优秀，这是这个成长型项目最宝贵的财富。

【主题】动态 NFT：NFT 的全面战争
【核心内容简介】动态 NFT 是通过实现 NFT 的元数据的可变性而赋予 NFT 其他功能，在视觉上可以根据背后元数据的流动而变化。
NFT 是部署在区块链上以及存储在智能合约中 的 tokenURI，包含 tokenID 和元数据两部分。ID 是一个系列每个 NFT 的编号，元数据包含该 NFT 的信息，如 NFT 描述、外部链接、图片地址、属性以及名称等。
动态 NFT 本质为实现元数据的改变。通过几种方法实现：1）通过智能合约读取链上以及链下的信息来进行条件判断，当符合条件时，则对 NFT 的元数据进行更新；2）模块化处理，将作品分为完整作品和图层两部分，一个 Master 可以由一个或多个 Layer 构成，且每个 Layer 都是相互独立的；3）为个人提供动态 NFT 定制的协议，帮助用户储存和表达 NFT 的元数据，同时可以更新 NFT 的元数据，来实现 NFT 的动态变化
【个人思考】NFT 可组合/可编程不仅可以增加 NFT 的应用场景，也可能是开启 NFT- Fi Summer 的必要条件。只有 NFT 之间的流动、可组性更强，才能带动对底层 NFT- Fi 基础设施的需求，如借贷、抵押等。同时，NFT 也有可能突破目前只是小图片的瓶颈，实现真正的可用。从传统行业的公司以 NFT 为踩点进入 Web3 的行为看，也许 NFT 也可以成为破圈的途径之一。当然，不希望原来炒作一个 NFT 变成炒作一小堆合成一个的 NFT。
